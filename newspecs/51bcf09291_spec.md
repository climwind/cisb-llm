# CISB Specification

## Vulnerability Description

**Source**
51bcf09291

**Description**
GCC optimizes away a store to iph->check before ip_fast_csum is called, because the inline assembly lacks a memory clobber, enabling reordering/elimination of the store and leading to incorrect IP header checksums.

**Evidence**
When compiling with GCC 3.4.2, the store 'iph->check = 0;' is eliminated before the call to ip_fast_csum, causing the checksum to be computed with stale data, resulting in bad IP fragment checksums and NFS over UDP failures.

**Requirement**
GCC 3.4.2 (or similar versions) with optimization enabled, SPARC architecture, inline assembly as defined in include/asm-sparc/checksum.h.

**Mitigation**
Add "memory" to the asm clobber list; e.g., change : "g2", "g3", "g4", "cc" to : "g2", "g3", "g4", "cc", "memory". Alternatively, use a compiler barrier or avoid inline assembly.
