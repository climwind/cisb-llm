# CISB Specification

## Vulnerability Description

**Source**
02828845dd

**Description**
GCC's -fdelete-null-pointer-checks optimization (enabled at -O2, -O3, -Os) treats inline asm memory operands as pointer dereferences, causing removal of subsequent null pointer checks. Developer expectation: prefetch macros do not dereference pointers, null checks should remain. Compiler behavior: assumes that if a pointer is used in an asm memory operand, it has already been dereferenced, so subsequent null checks are unnecessary.

**Evidence**
Code explicitly checks for null after a prefetch() macro. After optimization, the null check is eliminated, leading to unconditional execution of the guarded code and a null pointer dereference. This causes kernel oopses in constructs like hlist_for_each_entry().

**Requirement**
GCC with -fdelete-null-pointer-checks (enabled by default at -O2, -O3, -Os). Inline asm statement with a memory operand that references the pointer.

**Mitigation**
Convert the macro containing the inline asm to a static inline function, which prevents GCC from assuming the pointer is dereferenced in the caller. Alternatively, disable -fdelete-null-pointer-checks.
