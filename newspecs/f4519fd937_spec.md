# CISB Specification

## Vulnerability Description

**Source**
f4519fd937

**Description**
Compiler optimization may reorder memory reads, causing the DR (descriptor ready) bit to be read after other fields in a status message. This creates a race condition where other fields read earlier might have invalid values.

**Evidence**
The commit message states: 'Due to compiler optimization, it's possible that dr_bit... is read last'. The fix ensures the DR bit is read first, followed by a read memory barrier (rmb), before reading the rest of the status message.

**Requirement**
Compiler version with optimization enabled (e.g., -O2) that allows instruction scheduling reordering memory reads. Architecture with weak memory ordering (e.g., ARM) may exacerbate the issue.

**Mitigation**
Insert explicit read memory barrier (rmb()) after reading the DR bit and before reading other fields. Alternatively, use READ_ONCE() or volatile to prevent reordering.
