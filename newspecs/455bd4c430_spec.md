# CISB Specification

## Vulnerability Description

**Source**
455bd4c430

**Description**
ARM assembly memset implementation did not return the first argument in r0, causing corruption when GCC 4.7.2+ optimizations reused the return value.

**Evidence**
GCC 4.7.2+ optimizes assuming memset returns the first argument in r0, reusing r0 after the call. The ARM assembly memset did not preserve r0, leading to register/memory corruption and runtime crashes.

**Requirement**
GCC 4.7.2+ with optimization enabled; ARM architecture.

**Mitigation**
Modify the memset assembly to save the original pointer (r0) into ip at entry, use ip as base register for stores, and save/restore r8 to avoid corruption, ensuring r0 remains unchanged.
