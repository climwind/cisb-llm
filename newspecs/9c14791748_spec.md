# CISB Specification

## Vulnerability Description

**Source**
9c14791748

**Description**
In the Linux kernel's hns3 driver, the debugfs function hns3_dump_page_pool_info reads pages_state_hold_cnt without a compiler barrier. The compiler may optimize the read by reusing a stale register-cached value, causing incorrect output. The developer expected a fresh read from memory each time debugfs is queried.

**Evidence**
A plain read of a shared variable (pages_state_hold_cnt) is passed to sprintf. After compiler optimization (e.g., -O2), the compiler eliminates the memory load and reuses a previously cached value. When debugfs is read, the output shows a stale count instead of the current value, misleading debugging and security auditing.

**Requirement**
Compiler with optimizations enabled (e.g., GCC -O2, typical kernel build). No specific hardware/architecture required.

**Mitigation**
Use READ_ONCE() to force a volatile access and prevent compiler optimization from reusing stale values.
