# CISB Specification

## Vulnerability Description

**Source**
12051b318b

**Description**
MIPS kernel code declared mips_io_port_base as const, but modified it through a pointer cast removing const. Clang exploited this undefined behavior to optimize away the modification, causing boot failures. Developer expected the modification to work (as with GCC), but the C standard considers modifying a const object via cast undefined behavior, which allowed Clang to break it.

**Evidence**
Pre-patch, malta_defconfig failed to boot with Clang; post-patch (removing const qualifier) boots successfully. The set_io_port_base function used `*(unsigned long *)&mips_io_port_base = base;` which the compiler could legally eliminate due to UB.

**Requirement**
Clang compiler for MIPS architecture, optimization enabled (any -O1 or higher), malta_defconfig target.

**Mitigation**
Remove the const qualifier from the variable declaration and definition. In the fix, mips_io_port_base was changed from `const unsigned long` to `unsigned long`.
