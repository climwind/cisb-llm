# CISB Specification

## Vulnerability Description

**Source**
7185ad2672

**Description**
GCC optimization removes memset() calls on stack variables going out of scope, leaving cryptographic keys and intermediate states uncleared in memory.

**Evidence**
In sha256_transform, memset(W, 0, 64 * sizeof(u32)) is removed; in tgr160_final, memset(D, 0, TGR192_DIGEST_SIZE) is removed. The compiled binary omits these clearing instructions, leaving sensitive data on the stack.

**Requirement**
GCC with dead store elimination (e.g., -O1 or higher). The buffer must be a local variable whose lifetime ends immediately after the memset.

**Mitigation**
Replace memset with memzero_explicit() to act as a compiler barrier. Alternatively, use volatile-qualified writes or compiler flag -fno-builtin-memset.
