# CISB Specification

## Vulnerability Description

**Source**
1bd46782d0

**Description**
GCC optimizes IO memory access functions (memset_io, memcpy_fromio, memcpy_toio) by replacing calls to standard memset/memcpy with optimized versions that may introduce unaligned accesses, invalid for device mappings.

**Evidence**
The commit message states: 'GCCs optimisation may introduce unaligned accesses which are invalid for device mappings.' The patch creates separate assembly symbols (mmioset/mmiocpy) to prevent GCC from recognizing and optimizing these calls.

**Requirement**
ARM architecture; GCC compiler; optimization enabled (any level that performs built-in function replacement). The vulnerability arises when inline IO functions call standard memset/memcpy.

**Mitigation**
Use separate non-standard function names for IO memory operations so the compiler does not treat them as built-in and does not apply unsafe optimizations. Alternatively, disable specific optimizations with -fno-builtin-memset or similar, but that is less targeted.
