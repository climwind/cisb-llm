# CISB Specification

## Vulnerability Description

**Source**
16f5e707d6

**Description**
Compiler store tearing of a tagged pointer (pointer + flag bit) in page->mapping allows a concurrent reader to observe the pointer without the flag, leading to misinterpretation of page state, crashes, or data corruption.

**Evidence**
CPU0 writes page->mapping with a value combining an anon_vma pointer and PAGE_MAPPING_ANON flag. The compiler may split the store into separate pointer and flag writes. CPU1, performing a lockless rmap walk, reads page->mapping after seeing PageLRU set. If the read observes the pointer without the flag, the page is treated as a non-anonymous mapping, leading to invalid memory accesses and corruption.

**Requirement**
Compiler with default optimization behavior that permits store tearing (e.g., GCC without explicit atomic support). SMP system with concurrent lockless readers. No specific optimization flag required; -O2 or similar.

**Mitigation**
Use WRITE_ONCE() to perform an atomic store, preventing store tearing. Alternatively, protect all accesses to the field with a lock, or convert the field to an atomic type.
