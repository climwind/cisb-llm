# CISB Specification

## Vulnerability Description

**Source**
a45e92a599

**Description**
GCC 4.8 and 4.9 optimize away the .sin.sin_addr.s_addr initializer in a union vxlan_addr when .sa.sa_family is present, treating the union as initialized via the .sa member and zeroing out other members. The developer expected that initializing .sa.sa_family and .sin.sin_addr.s_addr would both be effective, but the compiler's union initialization semantics caused the address field to be zeroed, leading to incorrect netlink L3 miss messages.

**Evidence**
Source code initializes union vxlan_addr ipa with .sa.sa_family = AF_INET and .sin.sin_addr.s_addr = tip->daddr. After compilation with GCC 4.8/4.9, the .sin.sin_addr.s_addr initializer is optimized away, resulting in a zeroed IP address in netlink L3 miss messages. User space programs receive incorrect network state information.

**Requirement**
GCC 4.8 or 4.9, any optimization level that enables union initialization optimization (e.g., -O2). Linux kernel environment with VXLAN driver and VXLAN_F_L3MISS flag enabled.

**Mitigation**
Use the correct union member for initialization: change .sa.sa_family to .sin.sin_family for IPv4 and .sin6.sin6_family for IPv6, so that the compiler does not treat the union as initialized via the .sa member. Alternatively, use memset to zero the union before assigning individual fields, or use compiler flags like -fno-builtin-memset if applicable.
