# CISB Specification

## Vulnerability Description

**Source**
067e710b9a

**Description**
GCC 4.5 reorders an extended CP15 read (mrc instruction) above an is_smp() test due to missing 'memory' clobber in inline assembly, causing an undefined instruction abort on ARM1136 r0 cores and preventing boot.

**Evidence**
The read_cpuid_ext macro defined with asm volatile only clobbering 'cc' is used inside cache_ops_need_broadcast() after an if(is_smp()) guard. GCC 4.5 hoists the mrc instruction before the guard, so the CP15 read executes unconditionally. On ARM1136 r0, which lacks the extended CP15 register, the read triggers an undefined instruction exception, halting the boot with a kernel panic.

**Requirement**
GCC 4.5 (or other compilers that aggressively reorder volatile asm without a memory barrier), ARM architecture with CP15 coprocessor, kernel compiled with CONFIG_CPU_CP15 and -O2 optimization, running on ARM1136 r0 (or any core where the extended CP15 register is unsupported).

**Mitigation**
Add a 'memory' clobber to the inline asm statement to prevent reordering across the conditional check. Alternatively, use a standalone memory barrier (asm volatile("" ::: "memory")) before the sensitive operation, or ensure that the asm is not moved out of the conditional by using a proper compiler barrier intrinsic.
