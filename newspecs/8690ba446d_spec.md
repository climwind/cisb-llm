# CISB Specification

## Vulnerability Description

**Source**
8690ba446d

**Description**
The Linux kernel driver for ATI Mach64 CT graphics cards used an empty for-loop as a delay to stabilize the clock after switching to sclk. GCC 4.1.x and later optimized away this empty loop, removing the intended delay and causing system hangs/crashes on sparc64.

**Evidence**
The empty for-loop 'for (i=0;i<=0x1ffff;i++);' was removed by the compiler optimizer because it has no side effects. This caused the clock switch sequence to execute without the required stabilization delay, leading to unstable clock signals and ultimately a system hang on sparc64.

**Requirement**
GCC version 4.1.x or later with optimization (e.g., -O2) on sparc64 with CT-based mach64 graphics hardware.

**Mitigation**
Replace the empty loop with a real delay function like udelay() that has a visible side effect and will not be optimized away. Alternatively, insert a compiler barrier (e.g., asm volatile("" : : : "memory")) inside the loop, or declare the loop variable as volatile.
