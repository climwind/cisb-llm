# CISB Specification

## Vulnerability Description

**Source**
commit 0f20615d64

**Description**
The BPF_CORE_READ_BITFIELD() macro contains a switch statement with multiple case arms that each perform a direct memory load of a different size (u8, u16, u32, u64). Missing break statements cause fall-through. Clang may reorder CO-RE relocation expressions (BYTE_OFFSET/BYTE_SIZE) into each case arm, leading to the wrong memory load size being used. The developer expected each case to perform an independent load of the correct size and not fall through.

**Evidence**
BEFORE patch: Clang applied BYTE_OFFSET relocation 4 times across switch cases with mismatched sizes (u8, u16, u32, u64 all using the same relocation). AFTER patch: BYTE_OFFSET is applied once before the switch, and each case performs the correct size load.

**Requirement**
Clang compiler for BPF target with default optimizations (e.g., -O2) and CO-RE relocation support. The code uses __builtin_preserve_access_index or equivalent builtins that generate relocations.

**Mitigation**
Add break statements to each case arm and insert an asm volatile barrier (barrier_var()) before the switch to prevent Clang from reordering relocations.
