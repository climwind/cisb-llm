# CISB Specification

## Vulnerability Description

**Source**
Commit 455bd4c430; GCC 4.7.2+ Optimization Behavior

**Description**
ARM assembly memset implementation modified the return register (r0) during execution. GCC 4.7.2+ optimizations assume memset returns the first argument in r0 and reuse it post-call. The mismatch causes register corruption and crashes.

**Evidence**
Compiler reuses r0 after memset call assuming it holds the destination pointer. Assembly overwrites r0 via store instructions (e.g., stmgeia r0!). Runtime crashes occur when compiler relies on r0 containing the original pointer.

**Requirement**
GCC 4.7.2 or later, ARM architecture, optimization enabled, custom assembly memset implementation.

**Mitigation**
Ensure assembly implementations preserve the return register (save/restore r0), use compiler built-ins (-fno-builtin-memset), or normalize assembly to save argument to scratch register before modification.
