# CISB Specification

## Vulnerability Description

**Source**
Commit 51bcf09291

**Description**
GCC optimizes away memory stores preceding inline assembly when the assembly lacks a memory clobber, assuming the assembly does not read or write memory. This breaks cases where the assembly implicitly reads memory pointed to by arguments.

**Evidence**
Store to checksum field eliminated before inline assembly. Assembly reads memory via pointer argument. Missing memory clobber allowed compiler to assume no memory dependency.

**Requirement**
GCC compiler, optimization enabled, inline assembly without memory clobber accessing memory via pointers.

**Mitigation**
Add memory to the inline assembly clobber list to prevent reordering or elimination of memory accesses.
