# CISB Specification

## Vulnerability Description

**Source**
Linux kernel commit 7185ad2672, GCC optimization behavior

**Description**
GCC optimizes away memset() calls on stack variables going out of scope, leaving cryptographic keys and intermediate states uncleared in memory. Developer expects memset() to clear sensitive data, but compiler treats it as dead code since the variable is not read after the clear operation.

**Evidence**
Source code specifies memory clearing via memset() on local buffers (W, D arrays in SHA256, Tiger, Whirlpool), but compiled binary omits these instructions due to optimization passes. Sensitive crypto data remains in stack memory instead of being cleared.

**Requirement**
GCC compiler with optimization enabled (typically -O2 or higher), memset() on stack/local variables that go out of scope immediately after the call without subsequent reads

**Mitigation**
Replace memset() with memzero_explicit() or equivalent volatile-aware clearing function, or use compiler options like -fno-builtin-memset to prevent optimization
