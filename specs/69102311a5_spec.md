# CISB Specification

## Vulnerability Description

**Source**
69102311a5

**Description**
GCC optimizer under -O2 replaces conditional stores with unconditional speculative stores, bypassing lock-based synchronization and causing data races in shared memory objects like kernel ring buffers.

**Evidence**
Report states GCC 4.3.4 optimizer replaces conditional load/store with non-conditional one. Testcase segfaults with default optimization, passes with --param allow-store-data-races=0. Kernel manifestation includes inconsistent ring buffer state when NMI_LOG used without proper locking.

**Requirement**
GCC compiler (affected versions), optimization level -O2 or higher, default allow-store-data-races parameter enabled. Multi-threaded or interrupt context (kernel).

**Mitigation**
Add compiler flag --param allow-store-data-races=0 to KBUILD_CFLAGS. Use volatile for shared state or explicit memory barriers.
