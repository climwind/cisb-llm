# CISB Specification

## Vulnerability Description

**Source**
Commit 0f20615d64 (libbpf)

**Description**
The BPF_CORE_READ_BITFIELD macro uses a switch statement to select memory load size. Missing break statements cause fall-through. Clang optimizes by reordering BYTE_OFFSET and BYTE_SIZE relocations without a barrier, causing BYTE_OFFSET to be applied multiple times with mismatched load sizes instead of once before the switch.

**Evidence**
Before patch: BYTE_OFFSET relocation applied 4 times across switch cases with mismatched sizes. After patch: BYTE_OFFSET applied once to compute pointer, then correctly-sized loads. Security impact includes information disclosure and logic errors in BPF programs.

**Requirement**
Clang compiler, BPF target architecture, optimization enabled, CO-RE relocation support.

**Mitigation**
Add break statements to switch cases, use barrier_var() or asm volatile to prevent compiler reordering of address calculations.
