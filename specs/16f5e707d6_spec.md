# CISB Specification

## Vulnerability Description

**Source**
Linux Kernel commit 16f5e707d6

**Description**
Compiler store tearing on a tagged pointer assignment to a shared struct field allows concurrent readers to observe inconsistent state.

**Evidence**
Assignment page->mapping = (struct address_space *) anon_vma where anon_vma includes PAGE_MAPPING_ANON. Without WRITE_ONCE, compiler splits store.

**Requirement**
Compiler default optimization allowing store tearing. Multi-core concurrent access to the struct field.

**Mitigation**
Use WRITE_ONCE() or atomic store primitives for the assignment.
