# CISB Specification

## Vulnerability Description

**Source**
Linux kernel vxlan driver commit (a45e92a599), GCC 4.8/4.9 union initialization optimization bug

**Description**
GCC 4.8 and 4.9 incorrectly optimize union initializers when multiple members are accessed. When a union is initialized using one member (e.g., .sa.sa_family), the compiler treats the entire union as initialized via that member and optimizes away subsequent initializations of other members (e.g., .sin.sin_addr.s_addr), resulting in zeroed fields at runtime.

**Evidence**
Source code explicitly initializes both .sa.sa_family and .sin.sin_addr.s_addr in union vxlan_addr, but compiled binary shows zeroed IP address field. Netlink L3 miss messages contained zeroed IPs instead of actual missed IP addresses, observable in user space network monitoring tools.

**Requirement**
GCC 4.8 or 4.9 with optimization enabled, union type with overlapping members, initialization pattern mixing different union member accesses

**Mitigation**
Initialize all union fields through the same member path (use .sin.sin_family instead of .sa.sa_family when also initializing .sin.sin_addr.s_addr), or use memset followed by field assignment, or upgrade to GCC version with fixed union initialization semantics
