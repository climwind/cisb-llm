# CISB Specification

## Vulnerability Description

**Source**
ARM Linux Kernel commit 7801/1 (GCC 4.5 reordering issue)

**Description**
GCC 4.5 reorders an inline assembly block performing extended CP15 reads above a runtime is_smp() safety check. The developer expects the check to guard execution on unsupported hardware, but the compiler optimizes the order, causing undefined instruction aborts on ARM1136 r0 cores.

**Evidence**
Compiler reorders mrc instruction before the is_smp() test. On ARM1136 r0, executing the CP15 read without support triggers an undefined instruction abort, leading to boot failure. The fix adds a 'memory' clobber to prevent reordering.

**Requirement**
GCC 4.5, ARM architecture (specifically ARM1136 r0), default optimization levels allowing instruction scheduling.

**Mitigation**
Add 'memory' to the clobber list of the inline assembly to enforce ordering barriers, or use compiler-specific barrier intrinsics.
