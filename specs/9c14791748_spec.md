# CISB Specification

## Vulnerability Description

**Source**
Linux Kernel Commit 9c14791748 (net: hns3: fix one incorrect value of page pool info when queried by debugfs)

**Description**
Compiler optimization causes stale read of a shared variable accessed via debugfs. The compiler caches or reorders the read of a counter variable, leading to inconsistent values exposed to userspace. Developer expects a fresh memory read; compiler assumes single-threaded safety.

**Evidence**
Debugfs output shows incorrect values before fix. Fix applies READ_ONCE() to the variable access within a formatting call, forcing a fresh load from memory.

**Requirement**
Linux kernel environment, compiler optimization enabled, variable shared between concurrent contexts without read-side locking.

**Mitigation**
Use READ_ONCE() macro for reading shared variables where locking is not feasible, or ensure read access is protected by appropriate synchronization primitives.
