# CISB Specification

## Vulnerability Description

**Source**
Commit 8690ba446d (Linux Kernel)

**Description**
Compiler optimization eliminates empty loops intended as timing delays. Developers expect cycle consumption; compilers assume no side effects and remove the code.

**Evidence**
gcc-4.0.x preserved loop; gcc-4.1.x eliminated it. Resulted in hardware hang on sparc64 due to missing delay.

**Requirement**
GCC 4.1.x or later, optimization enabled (e.g., -O2), non-volatile loop counter.

**Mitigation**
Use explicit delay APIs (e.g., udelay), volatile qualifiers on loop counters, or compiler memory barriers.
