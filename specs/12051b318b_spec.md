# CISB Specification

## Vulnerability Description

**Source**
Commit 12051b318b

**Description**
A global variable is declared and defined as const but modified at runtime via pointer casting. Compilers exploiting const-correctness undefined behavior optimize away the modification, assuming immutability, which causes runtime failure such as boot failure.

**Evidence**
The set_io_port_base function casts the address of the const variable mips_io_port_base to a non-const pointer and writes to it. Clang assumes the const variable is immutable, ignores the write, and uses the initial value, leading to incorrect I/O port mapping and boot failure on malta_defconfig.

**Requirement**
Clang compiler or GCC with strict const optimization, optimization enabled, MIPS architecture or any platform relying on runtime update of the const variable.

**Mitigation**
Remove the const qualifier from the variable declaration and definition to permit legitimate modification without invoking undefined behavior.
