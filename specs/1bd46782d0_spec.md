# CISB Specification

## Vulnerability Description

**Source**
Linux kernel commit 1bd46782d08b01b73df0085b51ea1021b19b44fd

**Description**
GCC optimizes memset/memcpy calls within IO memory access wrappers, assuming aligned memory. IO memory mappings may not support unaligned accesses, leading to hardware faults. The fix uses distinct symbols (mmioset/mmiocpy) to prevent GCC from recognizing them as standard library functions eligible for optimization.

**Evidence**
Compiler replaces IO access function calls with optimized instructions assuming alignment. Runtime behavior changes from safe assembly implementation to unsafe optimized sequence. Hardware faults occur on device mappings.

**Requirement**
GCC compiler, ARM architecture, optimization enabled, IO memory usage.

**Mitigation**
Use distinct symbol names for IO memory operations that the compiler does not recognize as standard library functions (e.g., mmioset instead of memset), or use compiler flags like -fno-builtin for specific files.
