# CISB Specification

## Vulnerability Description

**Source**
Commit f4519fd937

**Description**
Compiler optimization reorders memory reads from a hardware status ring. The synchronization flag (DR bit) is read after dependent data fields instead of before, allowing processing of stale data if hardware updates concurrently.

**Evidence**
Report states compiler may read dr_bit last. Fix separates dr_bit read, places it before message copy, and adds rmb(). Digest shows original call sequence: wil_get_next_rx_status_msg followed by wil_rx_status_get_desc_rdy_bit.

**Requirement**
Compiler optimization enabled. Shared memory region (DMA/MMIO). Architecture allowing memory reordering without barriers.

**Mitigation**
Read synchronization flag first using volatile/READ_ONCE. Insert read memory barrier (rmb) before reading dependent data. Restructure code to enforce order.
